In Express, an API route is just a URL endpoint that sends or receives data — typically JSON — rather than HTML.(Application Programming Interface (API))

What is a RESTful API?
REST stands for Representational State Transfer.
Key Principles of RESTful APIs

Stateless – Each request contains all info the server needs.
Resource-based – URLs represent resources (users, posts, products, etc.).
Use proper HTTP methods – GET, POST, PUT, DELETE.
Return proper status codes – 200 OK, 201 Created, 404 Not Found, 500 Internal Server Error.
Use JSON – For sending and receiving data.(no html)

Url module(***** imp learn more)
---------
const { URL } = require('url');
const myURL = new URL('https://example.com:8080/path?name=John#section1');
console.log(myURL.hostname);  // 'example.com'
console.log(myURL.port);      // '8080'
console.log(myURL.pathname);  // '/path'
console.log(myURL.search);    // '?name=John'
console.log(myURL.hash);      // '#section1'

| Property                | Description             | Example Value                                                        |
| ----------------------- | ----------------------- | -------------------------------------------------------------------- |
| `href`                  | Full URL                | `'https://user:pass@sub.example.com:8080/p/a/t/h?query=string#hash'` |
| `origin`                | Protocol + Host         | `'https://sub.example.com:8080'`                                     |
| `protocol`              | URL scheme              | `'https:'`                                                           |
| `username` / `password` | Auth credentials        | `'user'` / `'pass'`                                                  |
| `host`                  | Host + Port             | `'sub.example.com:8080'`                                             |
| `hostname`              | Host only               | `'sub.example.com'`                                                  |
| `port`                  | Port number             | `'8080'`                                                             |
| `pathname`              | Path section            | `'/p/a/t/h'`                                                         |
| `search`                | Query string            | `'?query=string'`                                                    |
| `searchParams`          | Easy query manipulation | `myURL.searchParams.get('query')`                                    |
| `hash`                  | Fragment identifier     | `'#hash'`                                                            |

Atuntecation
------------
Stateful-which maintains data on server side
Stateless-which doesn't(JWT)

Stateful
--------
Enter username and password it gives session uid
Uid can be given through cookies,response,headers.
Cookies are just small pieces of data stored in your browser. Websites ask for them to improve your experience, track activity, or serve ads — not all cookies are for “spying.”

| Feature               | Cookie-based Auth                                        | Token-based Auth (JWT)                                          |
| --------------------- | -------------------------------------------------------- | --------------------------------------------------------------- |
| Storage               | Browser automatically stores cookies                     | Usually localStorage, sessionStorage, or in-memory              |
| Sent with request     | Automatically sent with every request to the same domain | Client must manually attach token in `Authorization` header     |
| Server state          | **Stateful**: server keeps session info                  | **Stateless**: server doesn’t store session (JWT contains info) |
| Scalability           | Harder for multiple servers (need shared session store)  | Easier for distributed systems (no server state)                |
| CSRF risk             | Higher if cookies not protected                          | Lower (if using Authorization header)                           |
| Mobile / cross-domain | Harder                                                   | Easier (token works anywhere)                                   |
